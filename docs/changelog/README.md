# Version Changelogs

Version changelogs are user-facing working notes for release notes. Keep one file per app version.

## Files

- `1.3.0.md` - current 1.3.0 tracking notes.
- `_template.md` - copy for future versions.

## Format

Use short bullets grouped by:

- Added
- Changed
- Fixed
- Removed
- Documentation and Release

Keep bullets simple and user-facing. Avoid internal refactors, test details, CI details, or code architecture notes unless they directly affect users.
