# Issue Labels

Recommended label set for public maintenance:

- `bug` - confirmed defect or broken behavior
- `enhancement` - new feature or improvement
- `documentation` - docs-only work
- `good first issue` - small, well-scoped contributor task
- `help wanted` - maintainer-approved external contribution welcome
- `needs repro` - report needs reproduction steps or sample files
- `area: app` - WPF app and UI behavior
- `area: core` - core package metadata or shared logic
- `area: audio` - audio parsing, conversion, loop metadata
- `area: packaging` - BEEmod package import/export flow
- `priority: high` - urgent or release-blocking work
- `priority: normal` - normal backlog priority
- `breaking-change` - changes compatibility, output format, or user workflow
